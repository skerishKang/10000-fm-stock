후보 저장·승격 흐름

1. 후보 JSON 보관

생성한 preview는 공식 데이터가 아니라 research workspace export 파일에 먼저 보관합니다.

  • research-workspace/exports/candidate-sources.template.json
  • research-workspace/exports/claim-candidates.template.json
  • research-workspace/exports/knowledge-note-candidates.template.json

2. 승격 검토

docs/promotion-checklist.md 기준으로 source, segment, claim, knowledge note 승격 여부를 검토합니다.

모호한 발언은 공식 claim으로 승격하지 말고 candidate로 유지하거나 reject합니다.

3. 검증 후 PR

공식 data/*.json 변경 후 반드시 로컬 validator를 실행합니다.

node scripts/validate-data.js

원본 PDF, 영상, 전체 자막, 전체 리포트 본문은 GitHub에 올리지 않습니다.
