# Knowledge 화면 설계

> **Issue**: #11
> **목적**: 산업/종목 교육용 지식 아카이브 화면 설계
> **상태**: Plan

---

## 1. 개요

Knowledge 화면은 **교육용 지식 아카이브**로서, 투자 판단의 근거가 되는 배경 지식을
체계적으로 학습하고 탐색할 수 있는 공간입니다.

**핵심 원칙:**
- Knowledge는 수익률 검증과 **분리된 가치**를 가집니다.
- 틀린 Claim이라도 배경 지식으로는 유용할 수 있습니다.
- 동일한 Source/Segment의 Claim과 연결 가능합니다.
- 사용자는 산업 → 종목 → 주제 순으로 탐색하며 깊이 있는 지식을 습득합니다.

---

## 2. 분류 체계

각 지식 노트는 다음 기준으로 분류됩니다.

| 분류 기준 | 설명 | 예시 |
|-----------|------|------|
| 산업 (Industry) | 상위 산업 분류 | 반도체, 2차전지, 바이오 |
| 종목 (Stock) | 개별 종목 연결 | 삼성전자, SK하이닉스 |
| 주제 (Topic) | 세부 학습 주제 | CAPEX 사이클, 공급망 리스크 |
| 난이도 (Difficulty) | basic / intermediate / advanced | - |
| 태그 (Tag) | 자유로운 연결 태그 | 밸류에이션, 매크로, 규제 |
| 출처 유형 (Source Type) | 리포트, 뉴스, 영상, 인터뷰 등 | - |
| 발언자 (Speaker) | 애널리스트, CFO, 교수 등 | - |
| 발행일 (Publish Date) | 원본 출처 발행일 | - |
| 추가일 (Added Date) | 시스템 등록일 | - |

## 3. 지식 노트 카드 (Knowledge Note Card)

Knowledge Note 하나의 데이터 구조입니다.

### 3.1 필드 구성

| 필드 | 필수 | 설명 |
|------|------|------|
| id | Y | 고유 식별자 |
| industry | Y | 산업 분류 |
| stock | N | 관련 종목 (선택) |
| topic | Y | 주제 |
| difficulty | Y | 난이도 (basic/intermediate/advanced) |
| summary | Y | 2~3문장 요약 |
| key_points | Y | 핵심 포인트 2~5개 |
| tags | N | 태그 배열 |
| source_title | Y | 출처 제목 |
| source_type | Y | 출처 유형 |
| source_url | N | 출처 링크 |
| source_segment | N | 영상 구간 / 페이지 번호 |
| speaker | N | 발언자 |
| publish_date | N | 발행일 |
| added_date | Y | 추가일 |
| related_claims | N | 연결된 Claim ID 배열 |

### 3.2 난이도 기준

| 난이도 | 대상 | 설명 |
|--------|------|------|
| basic | 초보자 | 업계 기본 개념, 용어 설명, 시장 구조 |
| intermediate | 중급자 | 비즈니스 모델, 경쟁 구도, 가치 사슬 |
| advanced | 고급자 | 고급 분석, 업사이드/리스크 시나리오, 딥다이브 |

### 3.3 카드 UI 예시

```
+--------------------------------------------------+
|  산업 > 반도체  |  난이도: intermediate        |
+--------------------------------------------------+
| 제목: 메모리 반도체의 슈퍼사이클과 CAPEX 투자     |
|                                                    |
| 요약:                                                |
| 메모리 반도체 시장은 수요-공급 불일치로 인해        |
| 주기적인 슈퍼사이클이 발생합니다. ...               |
|                                                    |
| 핵심 포인트:                                        |
| - 슈퍼사이클은 평균 2~3년 지속                       |
| - CAPEX 증가는 공급 과잉 신호일 수 있음              |
| - HBM 수요가 사이클 구조 변화 중                     |
|                                                    |
| 태그: #메모리 #CAPEX #사이클 #HBM                   |
| 출처: KB증권 산업분석 리포트 (2024-03) p.12         |
| 발언자: 김xx 애널리스트                             |
+--------------------------------------------------+
```

## 4. 산업별 페이지 (Industry Detail)

`industry-detail.html?industry=semiconductor`

### 4.1 페이지 구성

| 섹션 | 설명 |
|------|------|
| 산업 개요 | 산업 설명, 시장 규모, 성장률 |
| 핵심 개념 | 해당 산업의 핵심 용어/개념 목록 (Knowledge Note 링크) |
| 관련 종목 | 산업 내 주요 종목 리스트 + 간략 정보 |
| 리스크 | 산업 고유 리스크 요인 |
| 투자 논리 | 산업 투자 시 고려할 핵심 논리 구조 |
| 관련 Claim | 같은 산업의 Claim 연결 |
| 관련 Source | 같은 산업의 Source 연결 |
| 관련 Expert | 산업 전문가 목록 |

### 4.2 확장성

- 산업 분류는 계층적 확장 가능 (대분류 > 중분류 > 소분류)
- 신규 산업 추가 시 config/industries.json 에 정의 추가
- 각 산업 페이지는 Knowledge Note 필터링 기본값으로 사용

### 4.3 산업 예시

```
+--------------------------------------------------+
|  반도체 (Semiconductor)                          |
| 산업 분류: IT > 전자부품 > 반도체                  |
+--------------------------------------------------+
| 요약: 글로벌 반도체 시장은 2024년 약 6천억 달러... |
|                                                    |
| 핵심 개념:                                         |
| - 파운드리 vs IDM vs 팹리스                        |
| - 노드 (나노미터) 경쟁                             |
| - HBM과 AI 반도체                                  |
|                                                    |
| 주요 종목: 삼성전자, SK하이닉스, 엔비디아, AMD     |
| 리스크: 지정학적 리스크, 중국 의존도, CAPEX 부담   |
+--------------------------------------------------+
```

## 5. 종목별 페이지 (Stock Detail)

### 5.1 페이지 구성

| 섹션 | 설명 |
|------|------|
| 종목 기본 정보 | 종목명, 티커, 산업, 시가총액 |
| 관련 Claim | 종목 관련 Claim 리스트 (긍정/부정 비율 포함) |
| Knowledge Note | 종목 관련 지식 노트 목록 |
| 긍정/부정 비율 | 시각화 (파이 차트 또는 바 차트) |
| 투자 포인트 | 핵심 투자 근거 정리 |
| 리스크 | 종목 고유 리스크 |

### 5.2 긍정/부정 비율

- 같은 종목의 Claim을 긍정(positive) / 부정(negative)으로 분류
- 비율은 시각화하여 한눈에 파악 가능
- Knowledge Note와 Claim의 연결을 통해 맥락 이해

### 5.3 종목 예시

```
+--------------------------------------------------+
|  삼성전자 (005930)                              |
| 산업: 반도체 and 가전  |  시총: 400조           |
+--------------------------------------------------+
| Claim 분포: [긍정 65%] [부정 35%]              |
| [--------------------.....]                    |
|                                                    |
| Knowledge Notes (5건):                             |
| - 메모리 반도체 슈퍼사이클과 CAPEX 투자 [intermediate]
| - HBM 시장에서의 삼성전자 경쟁력 [advanced]        |
| - 파운드리 사업 전망과 위기 [intermediate]          |
| - 삼성전자 밸류에이션 분석 [basic]                 |
| - DS부문 조직 개편의 의미 [intermediate]            |
|                                                    |
| 투자 포인트:                                       |
| - HBM 시장 점유율 확대 가능성                      |
| - 파운드리 수주 증가 추세                          |
|                                                    |
| 리스크:                                            |
| - HBM 기술 격차 (SK하이닉스 대비)                   |
| - 파운드리 수율 이슈                               |
+--------------------------------------------------+
```

## 6. 수익률 검증과의 관계

Knowledge 화면은 **수익률 검증과 분리**되어 있지만, **연결**될 수 있습니다.

### 6.1 분리 원칙

| 구분 | Claim | Knowledge Note |
|------|-------|----------------|
| 목적 | 예측 수익률 검증 | 배경 지식 학습 |
| 검증 대상 | O | X |
| 핵심 가치 | 맞힘/틀림 여부 | 이해도 향상 |
| 수명 | 제한적 (시장 변화) | 반영구적 |

### 6.2 연결 방식

- Knowledge Note와 Claim은 **같은 Source + Segment**를 공유할 경우 연결
- 연결된 Claim이 틀렸더라도 Knowledge Note의 가치는 유지
- 사용자는 Claim의 검증 결과를 참고하여 Knowledge Note의 맥락을 이해

### 6.3 시나리오 예시

> **예시 1**: "2024년 HBM 시장 3배 성장"이라는 Claim이 있었고,
> 실제로 2.8배 성장해서 "거의 맞힘" 판정.
> 이 Claim의 배경 Knowledge Note인 "HBM 시장 구조와
> AI 수요 분석"은 여전히 유효한 교육 자료로 활용 가능.

> **예시 2**: "삼성전자 파운드리 점유율 20% 달성" Claim이
> "틀림" 판정을 받았어도, "파운드리 산업의 비즈니스 모델"이라는
> Knowledge Note는 여전히 유용한 학습 자료.

## 7. 검색 및 필터링

### 7.1 검색 기능

- 키워드 검색 (제목, 요약, 핵심 포인트)
- 태그 기반 검색 (#태그)
- 전체 텍스트 검색

### 7.2 필터 옵션

| 필터 | 옵션 |
|------|------|
| 산업 | 전체, 반도체, 2차전지, 바이오, ... |
| 종목 | 전체 또는 특정 종목 |
| 난이도 | basic / intermediate / advanced |
| 태그 | 다중 선택 |
| 출처 유형 | 리포트, 뉴스, 영상, 인터뷰 |
| 발행일 | 기간 지정 |
| 추가일 | 기간 지정 |

### 7.3 정렬 옵션

- 추가일순 (최신순)
- 발행일순
- 난이도순
- 관련 Claim 수순

### 8.1 KnowledgeNote

```json
{
  "id": "KN-001",
  "industry": "semiconductor",
  "stock": "005930",
  "topic": "메모리 반도체 사이클",
  "difficulty": "intermediate",
```

## 9. 향후 확장 계획

### 9.1 단계별 로드맵

| 단계 | 내용 |
|------|------|
| Phase 1 | Knowledge Note CRUD, 기본 카드 UI, 산업/종목 연결 |
| Phase 2 | 산업별 페이지, 종목별 페이지, 검색/필터링 |
| Phase 3 | Claim 연결, 수익률 검증 결과 표시 |
| Phase 4 | 학습 진행도, 북마크, 퀴즈 기능 |

### 9.2 추가 고려 사항

- Knowledge Note 간의 연관 관계 그래프 시각화
- 학습 히스토리 기반 추천 시스템
- 사용자 기여 (Knowledge Note 제안/수정)
- 오프라인 학습 지원 (PDF Export)
