## 1. 개요

본 문서는 FM-Stock 플랫폼에서 각 **전문가(인물)** 의 신뢰도와 전문성을 데이터 기반으로 평가할 수 있는 상세 페이지 설계를 정의합니다.

### 1.1 배경

기존 플랫폼에서는 개별 전문가의 발언 이력과 성과를 한눈에 파악할 수 있는 전용 페이지가 부재했습니다. 사용자가 특정 인물의 **적중률, 평균 수익률, 산업별 강점** 등을 직관적으로 확인할 수 있도록 하여 정보 비대칭을 해소하고 투자 판단의 질을 높이는 것이 목표입니다.

### 1.2 목표

- 각 전문가의 **검증 가능한 발언 성과**를 집계하여 신뢰도 지표로 제시
- **산업별, 종목별 강점/약점**을 시각화하여 전문 분야를 한눈에 파악
- **교육용 지식 노트** 연동으로 전문가의 지식 기여도 측정
- 사용자가 특정 인물의 발언을 **시간순 히스토리**로 추적 가능

### 1.3 대상 페르소나

| 페르소나 | 니즈 |
|---------|------|
| 개인 투자자 | 믿을 수 있는 전문가를 찾아 참고하고 싶다 |
| 애널리스트 자신 | 자신의 성과를 객관적으로 검증받고 싶다 |
| 플랫폼 운영자 | 전문가 인센티브 및 랭킹 시스템의 기초 데이터로 활용 |

## 4. 화면 구성 섹션 상세

### 4.1 인물 헤더

**목적:** 해당 인물의 신원과 기본 정보를 표시하여 신뢰성과 정체성을 확립한다.

**표시 항목:**

| 항목 | 표시 방식 | 예시 |
|------|----------|------|
| 이름 | 큰 텍스트 + 뱃지 | 홍길동 |
| 표시이름 | 부제목 | [Trophy] 애널리스트 홍길동 |
| 유형 | 아이콘 + 레이블 | [Satellite Antenna] 방송 출연 / [Chart] 애널리스트 |
| 소속 | 텍스트 | OO증권 리서치센터 |
| 주요 산업 | 태그 목록 | 반도체 . 2차전지 . 자동차 |
| 주요 종목 | 태그 목록 | 삼성전자 . SK하이닉스 |
| 외부 링크 | 아이콘 링크 | [YouTube] [블로그] [트위터] |
| 설명 | 짧은 소개글 | "20년 경력의 반도체 전문 애널리스트..." |

**UX 고려사항:**
- 유형별로 다른 아이콘과 색상을 사용하여 시각적 구분
- 외부 링크는 새 탭에서 열리도록 처리
- 설명은 2줄 이상일 경우 "더보기" 접이식 처리

### 4.2 성과 요약

**목적:** 인물의 전체 발언 성과를 수치화된 KPI로 한눈에 파악한다.

**KPI 카드 리스트:**

| 지표 | 설명 | 계산식 |
|------|------|--------|
| 전체 발언 수 | 총 claim 수 | COUNT(claims) |
| 검증 완료 수 | evaluation이 완료된 claim 수 | COUNT(evaluations WHERE judgment != pending) |
| 검증 대기 수 | 아직 검증되지 않은 발언 수 | COUNT(claims) - COUNT(verified) |
| 적중률 | 맞춘 발언의 비율 | (COUNT(hits) / COUNT(verified)) x 100 |
| 평균 수익률 | 검증된 발언의 평균 수익률 | AVG(return_rate) |
| 중앙값 수익률 | 수익률의 중앙값 | MEDIAN(return_rate) |
| 평균 초과수익률 | 알파(alpha)의 평균 | AVG(alpha) |
| 목표가 도달률 | 목표가를 달성한 비율 | (COUNT(target_price_achieved=true) / COUNT(verified)) x 100 |
| 평균 검증 기간 | 발언일->검증일 평균 일수 | AVG(DATEDIFF(verified_date, statement_date)) |

**UX 고려사항:**
- 숫자는 콤마 포맷팅 및 적절한 소수점 자리수 표시
- 적중률은 색상 그라데이션 (높을수록 초록, 낮을수록 빨강)
- 호버 시 상세 설명 툴팁 제공
- 로딩 중에는 스켈레톤 UI 사용

### 4.3 산업별 강점/약점

**목적:** 인물이 어떤 산업 분야에서 강점을 보이는지 데이터로 증명한다.

**표시 방식:** 테이블 + 막대 차트 (선택 사항)

| 산업 | 검증 수 | 적중률 | 평균 알파 | 강점 표시 |
|------|---------|--------|-----------|----------|
| 반도체 | 22 | 68.2% | +8.1% | [Star] 강점 |
| 2차전지 | 15 | 73.3% | +5.6% | [Star] 강점 |
| 자동차 | 8 | 50.0% | -1.2% | [Warning] 약점 |
| 바이오 | 3 | 33.3% | -3.5% | [Warning] 약점 |

**분류 기준:**

| 조건 | 판정 |
|------|------|
| 검증 수 >= 5 AND 적중률 > 60% | [CheckMark] 강점 |
| 검증 수 >= 5 AND 적중률 < 45% | [Warning] 약점 |
| 검증 수 < 5 | [Info] 데이터 부족 |
| 그 외 | [Minus] 보통 |

**UX 고려사항:**
- 검증 수가 적은 산업은 회색 처리 또는 데이터 부족 문구 표시
- 정렬 가능 (검증 수/적중률/알파 기준)
- 모바일에서는 가로 스크롤 테이블

### 4.4 발언 히스토리 (표)

**목적:** 인물의 모든 발언을 시간순으로 추적하고 검증 결과를 확인한다.

**테이블 컬럼:**

| 컬럼 | 설명 | 예시 |
|------|------|------|
| 날짜 | 발언 일자 | 2025-01-15 |
| 출처 | 발언 매체/채널 | OO경제TV |
| 구간/페이지 | 발언 시점의 페이지 또는 구간 | p.12 / 3분기 |
| 종목 | 대상 종목명 | 삼성전자 |
| 산업 | 종목의 산업 분류 | 반도체 |
| 방향성 | 상승/하락/중립 아이콘 | [GreenCircle] 상승 |
| 기준가 | 발언 당시 가격 | 73,500원 |
| 검증가 | 검증 시점 가격 | 81,200원 |
| 수익률 | (검증가-기준가)/기준가 | +10.5% |
| 알파 | 초과 수익률 | +3.2% |
| 판정 | hit/miss/pending | [CheckMark] HIT |

**UX 고려사항:**
- 기본 정렬: 날짜 내림차순
- 필터 옵션: 판정별(hit/miss/pending), 산업별, 방향성별
- 검색: 종목명 또는 출처로 검색
- 페이지네이션 (1페이지당 20행)
- 반응형: 모바일에서는 주요 컬럼만 표시 (날짜, 종목, 수익률, 판정)
- 판정 뱃지 색상: hit=초록, miss=빨강, pending=회색

## 5. 계산 방식

### 5.1 데이터 수집 흐름

```
1. expertId 수신
2. expert 테이블에서 기본 정보 조회
3. claim 테이블에서 expertId로 모든 발언 조회
4. evaluation 테이블에서 각 claim_id로 검증 결과 조회
5. knowledge_note 테이블에서 expertId로 노트 목록 조회
6. 검증된 claim 기준으로 각종 지표 계산
```

### 5.2 지표별 계산 로직

**적중률:**
```python
def calculate_hit_rate(evaluations):
    total_verified = len([e for e in evaluations if e.judgment != 'pending'])
    hits = len([e for e in evaluations if e.judgment == 'hit'])
    return round((hits / total_verified * 100), 1) if total_verified > 0 else 0
```

**평균 수익률:**
```python
def calculate_avg_return(evaluations):
    verified = [e for e in evaluations if e.judgment != 'pending']
    if not verified:
        return 0
    return round(sum(e.return_rate for e in verified) / len(verified), 2)
```

**중앙값 수익률:**
```python
def calculate_median_return(evaluations):
    verified_returns = sorted([e.return_rate for e in evaluations if e.judgment != 'pending'])
    n = len(verified_returns)
    if n == 0:
        return 0
    mid = n // 2
    if n % 2 == 0:
        return round((verified_returns[mid-1] + verified_returns[mid]) / 2, 2)
    return round(verified_returns[mid], 2)
```

**평균 초과수익률 (알파):**
```python
def calculate_avg_alpha(evaluations):
    verified = [e for e in evaluations if e.judgment != 'pending']
    if not verified:
        return 0
    return round(sum(e.alpha for e in verified) / len(verified), 2)
```

**목표가 도달률:**
```python
def calculate_target_achievement_rate(evaluations):
    verified = [e for e in evaluations if e.judgment != 'pending']
    if not verified:
        return 0
    achieved = len([e for e in verified if e.target_price_achieved])
    return round((achieved / len(verified) * 100), 1)
```

**산업별 통계:**
```python
def calculate_industry_stats(claims, evaluations):
    industry_data = {}
    for claim, eval in zip(claims, evaluations):
        industry = claim.industry
        if industry not in industry_data:
            industry_data[industry] = {'count': 0, 'hits': 0, 'alphas': []}
        industry_data[industry]['count'] += 1
        if eval.judgment == 'hit':
            industry_data[industry]['hits'] += 1
        if eval.judgment != 'pending':
            industry_data[industry]['alphas'].append(eval.alpha)
    # 각 산업별 적중률, 평균 알파 계산
    ...
```
