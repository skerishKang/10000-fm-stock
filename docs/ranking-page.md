# Ranking 화면 설계 문서

> **이슈**: #13  
> **목적**: 전문가/발언/산업별 성과 랭킹 화면 설계  
> **핵심 방향**: 랭킹은 단순한 줄 세우기가 아니라, 흥미 요소이자 전문가 검증 기능으로서 다양한 기준을 제공한다.

---

## 1. 개요

본 문서는 FM-Stock 서비스의 랭킹 페이지 설계를 정의한다. 랭킹 페이지는 사용자에게 투자 판단의 참고자료를 제공하고, 전문가의 발언 성과를 투명하게 공개하며, 커뮤니티 내 지식 기여를 장려하는 것을 목표로 한다. 단순히 수익률 순위만 나열하는 것을 지양하고, 여러 기준(수익률, 초과수익률, 적중률, 산업별 성과, 지식 기여 등)을 복합적으로 제공하여 정보의 다양성과 신뢰성을 확보한다.

### 1.1. 랭킹의 목적

랭킹 페이지는 다음 5가지 질문에 답할 수 있도록 설계된다.

1. **어떤 발언이 가장 높은 수익률을 기록했는가?**  
   - 개별 발언의 절대 수익률 기준 랭킹. 사용자가 과거 고수익 발언을 탐색하고, 해당 발언자의 전략을 학습할 수 있도록 돕는다.

2. **어떤 발언이 시장 대비 가장 좋은 초과수익률을 기록했는가?**  
   - 벤치마크(KOSPI, KOSDAQ, S&P 500 등) 대비 초과수익률 기준 랭킹. 단순 시장 흐름에 편승한 발언과 진정한 알파를 창출한 발언을 구별한다.

3. **어떤 전문가가 꾸준히 잘 맞췄는가?**  
   - 전문가별 평균 초과수익률과 적중률을 복합적으로 평가. 일회성 고수익보다 꾸준한 성과를 낸 전문가를 발굴한다.

4. **특정 산업에서 누가 강한가?**  
   - 반도체, 2차전지, 조선, 바이오, AI, 로봇 등 주요 산업별로 전문가 랭킹을 제공하여, 산업에 특화된 전문가를 찾을 수 있게 한다.

5. **지식을 많이 제공한 사람은 누구인가?**  
   - Knowledge Note 등록 수 기준 랭킹으로, 커뮤니티에 기여하는 전문가를 시각화하고 지식 생태계를 활성화한다.

## 2. 6가지 랭킹 종류

### 2.1. 발언 수익률 TOP

| 항목 | 설명 |
|------|------|
| **정의** | 검증이 완료된 개별 발언 중 절대 수익률이 높은 순서대로 정렬 |
| **표시 항목** | 순위, 발언자(전문가명), 종목명, 산업, 발언일, 검증기간, 수익률(%), 초과수익률(%), 출처 |
| **검증 조건** | 검증 상태가 '완료'이고, 검증 기간(minimum 7일) 이상 경과한 발언만 포함 |
| **정렬 기본값** | 수익률 내림차순 |
| **표본 제한** | 초기에는 전체 발언 대상이나, 추후 최소 보유 기간(30일) 조건 추가 검토 |
| **필터 옵션** | 산업 필터, 기간 필터(최근 1개월/3개월/6개월/1년/전체), 자산 유형 필터(주식/ETF/선물 등) |

**화면 구성 예시**:

```
순위 | 발언자 | 종목 | 산업 | 발언일 | 검증기간 | 수익률 | 초과수익률 | 출처
1    | 김OO   | 삼성전자 | 반도체 | 2026-01-15 | 90일 | +48.2% | +32.1% | 오전전략회의
2    | 이OO   | 에코프로 | 2차전지 | 2026-02-03 | 60일 | +41.5% | +28.7% | 데일리브리핑
3    | 박OO   | HLB    | 바이오 | 2026-03-10 | 45일 | +35.8% | +22.4% | 마감코멘트
...
```

### 2.2. 발언 초과수익률 TOP

| 항목 | 설명 |
|------|------|
| **정의** | 벤치마크(시장 지수) 대비 초과수익률이 높은 순서대로 정렬 |
| **표시 항목** | 순위, 발언자, 종목, 산업, 발언일, 검증기간, 수익률, 초과수익률, 벤치마크, 출처 |
| **벤치마크 설정** | KOSPI (국내 주식), KOSDAQ (코스닥), S&P 500 (미국 주식), 산업 지수(반도체: KIS 반도체 지수 등) |
| **의의** | 시장 베타 효과를 제거하고 실제 분석 역량에 기반한 성과 측정 가능 |
| **최소 표본** | 개별 발언이므로 별도의 최소 표본 조건 없음 |

**초과수익률 계산식**:

```
초과수익률 = 발언 수익률 - 해당 기간 벤치마크 수익률
```

### 2.3. 전문가 평균 초과수익률 TOP

| 항목 | 설명 |
|------|------|
| **정의** | 전문가(expertId)별 모든 검증 완료 발언의 평균 초과수익률 순위 |
| **표시 항목** | 순위, 전문가명, 소속, 평균 초과수익률(%), 표본 수(검증 발언 수), 평균 수익률(%), 최고 수익률(%), 최저 수익률(%) |
| **최소 표본 수** | 초기 5개 이상의 검증 발언이 있는 전문가만 포함 |
| **표본 수 표시 원칙** | 반드시 표본 수를 함께 표시 (예: "검증 발언 42개, 적중률 61.9%, 평균 수익률 +14.2%, 평균 초과수익률 +6.8%") |
| **신뢰도 표시** | 표본 수에 따른 신뢰도 표시 (5-9개: 낮음, 10-19개: 중간, 20개 이상: 높음) |

**왜곡 방지 장치**:
- 표본 수가 적은 전문가는 랭킹 상단에 노출되더라도 신뢰도가 낮음을 명시적으로 표시
- 5개 미만인 경우 랭킹에서 제외 (초기 기준, 추후 조정 가능)
- 종합 점수는 제공하지 않고 기준별 랭킹만 제공

**화면 구성 예시**:

```
순위 | 전문가 | 소속 | 평균 초과수익률 | 표본 수 | 평균 수익률 | 신뢰도
1    | 김OO   | OO증권 | +8.2%          | 23개    | +15.3%      | 🟢 높음
2    | 이OO   | OO자산운용 | +6.7%       | 18개    | +12.1%      | 🟡 중간
3    | 최OO   | OO리서치 | +5.9%         | 42개    | +11.8%      | 🟢 높음
...
```

### 2.4. 전문가 적중률 TOP

| 항목 | 설명 |
|------|------|
| **정의** | 전문가별 발언의 적중률(hit rate) 순위 |
| **적중 기준** | hit: 목표가 도달 또는 방향성 정확 (+1점) |
| | partial_hit: 목표가에 근접했으나 미달 (+0.5점) |
| | miss: 방향성 틀림 (+0점) |
| **계산식** | `적중률 = (hit 수 + partial_hit 수 * 0.5) / 전체 검증 발언 수 * 100` |
| **표시 항목** | 순위, 전문가명, 소속, 적중률(%), hit 수, partial_hit 수, miss 수, 표본 수 |
| **최소 표본 수** | 초기 5개 이상 |
| **추가 고려사항** | 단순 적중률만으로 평가 시 발생할 수 있는 편향(예: 적중률은 높으나 수익률은 낮은 경우)을 보완하기 위해, 적중률 랭킹 옆에 평균 수익률을 함께 표시 |

### 2.5. 산업별 전문가 랭킹

| 항목 | 설명 |
|------|------|
| **정의** | 특정 산업 내에서 검증된 발언을 기준으로 전문가 랭킹 제공 |
| **대상 산업** | 반도체, 2차전지, 조선, 바이오, AI, 로봇, 자동차, 금융, 인터넷/플랫폼, 헬스케어, 화학, 에너지, 게임, 엔터테인먼트, 건설, 유통 등 |
| **정렬 기준** | 산업 내 평균 초과수익률 (기본값), 산업 내 적중률 (옵션) |
| **표시 항목** | 순위, 전문가명, 소속, 해당 산업 내 검증 발언 수, 평균 수익률, 평균 초과수익률, 적중률 |
| **최소 표본 수** | 해당 산업 내 3개 이상의 검증 발언 |
| **특화 지수** | 전체 대비 해당 산업 발언 비율이 50% 이상인 전문가에게 '산업 전문가' 배지 표시 |

**화면 구성 예시** (반도체 산업 선택 시):

```
산업: 반도체 (전체 전문가: 24명)

순위 | 전문가 | 소속 | 검증 발언 | 평균 수익률 | 평균 초과수익률 | 적중률 | 배지
1    | 김OO   | OO증권 | 15개     | +18.5%      | +9.2%           | 73.3%  | 🏅 산업 전문가
2    | 박OO   | OO자산운용 | 8개  | +14.2%      | +7.1%           | 62.5%  | -
3    | 이OO   | OO리서치 | 22개   | +12.8%      | +6.5%           | 68.2%  | 🏅 산업 전문가
...
```

**산업 선택 UI**:
- 드롭다운 또는 탭 형태로 주요 산업을 선택
- 산업 검색 기능 포함
- 산업별 전문가 수 함께 표시

### 2.6. 지식 기여 랭킹

| 항목 | 설명 |
|------|------|
| **정의** | Knowledge Note 등록 수 기준 전문가 랭킹 |
| **표시 항목** | 순위, 전문가명, 소속, Knowledge Note 수, 총 조회수, 총 좋아요 수, 최근 등록일 |
| **의의** | 지식 공유를 장려하고, 커뮤니티 내 영향력 있는 전문가를 발굴 |
| **정렬 옵션** | Knowledge Note 수 (기본), 조회수 순, 좋아요 순 |
| **최소 조건** | 1개 이상의 Knowledge Note 등록 시 자동 포함 |
| **추가 기능** | 전문가 클릭 시 해당 전문가의 Knowledge Note 목록으로 이동 |

### 3.2. 공통 필터

모든 랭킹 탭에 공통으로 적용되는 필터:

| 필터 | 옵션 | 기본값 |
|------|------|--------|
| 산업 | 전체, 반도체, 2차전지, 조선, 바이오, AI, 로봇 등 | 전체 |
| 기간 | 1개월, 3개월, 6개월, 1년, 전체 | 3개월 |
| 발언 유형 | 전체, 매수/매도 의견, 목표가 제시, 시황 분석 | 전체 |
| 검증 기간 | 7일 이상, 30일 이상, 90일 이상 | 30일 이상 |

### 3.3. 데이터 표시 규칙

1. **숫자 포맷**: 수익률/초과수익률은 소수점 첫째 자리까지 표시 (예: +14.2%, -3.5%)
2. **색상 코드**: 양수는 빨간색(상승), 음수는 파란색(하락) — 한국 주식 시장 관행 준수
3. **링크**: 전문가명 클릭 → 전문가 상세 페이지, 종목명 클릭 → 종목 상세 페이지
4. **툴팁**: 각 항목에 마우스 오버 시 상세 설명 툴팁 제공
5. **반응형**: 데스크톱(테이블 뷰)과 모바일(카드 뷰) 모두 지원

### 3.4. 새로고침 정책

| 데이터 | 갱신 주기 | 방식 |
|--------|-----------|------|
| 발언 수익률/초과수익률 | 실시간 (검증 완료 시) | 웹소켓 푸시 또는 주기적 폴링 |
| 전문가 평균 초과수익률 | 1일 1회 (야간 배치) | Redis 캐싱 후 제공 |
| 전문가 적중률 | 1일 1회 (야간 배치) | Redis 캐싱 후 제공 |
| 산업별 랭킹 | 1일 1회 (야간 배치) | Redis 캐싱 후 제공 |
| 지식 기여 랭킹 | 실시간 (Knowledge Note 등록 시) | 웹소켓 푸시 |

## 4. 데이터 모델

### 4.1. 랭킹 데이터 스키마 (예시)

```typescript
interface RankingEntry {
  rank: number;
  type: RankingType; // 'statement_return' | 'statement_excess' | 'expert_avg_excess' | 'expert_hit_rate' | 'industry_expert' | 'knowledge_contribution'
  expertId: string;
  expertName: string;
  affiliation?: string;
  industry?: string;
  metrics: {
    returnRate?: number;       // 수익률 (%)
    excessReturnRate?: number; // 초과수익률 (%)
    hitRate?: number;          // 적중률 (%)
    hitCount?: number;
    partialHitCount?: number;
    missCount?: number;
    sampleCount: number;       // 표본 수 (필수)
    knowledgeNoteCount?: number;
    totalViews?: number;
    totalLikes?: number;
  };
  benchmark?: string;          // 벤치마크 지수명
  statementId?: string;        // 발언 수익률 랭킹의 경우
  stockName?: string;          // 발언 수익률 랭킹의 경우
  statementDate?: string;      // 발언일
  verificationPeriod?: number; // 검증 기간 (일)
  source?: string;             // 출처
  reliability: 'low' | 'medium' | 'high';
}
```

### 4.2. API 설계 (초안)

```
GET /api/v1/rankings/statement-return
  ?industry={industry}
  &period={period}
  &statementType={type}
  &verificationMinDays={days}
  &page={page}
  &limit={limit}

GET /api/v1/rankings/statement-excess-return
  ?industry={industry}
  &period={period}
  &benchmark={benchmark}
  &page={page}
  &limit={limit}

GET /api/v1/rankings/expert-avg-excess-return
  ?minSamples={minSamples}
  &page={page}
  &limit={limit}

GET /api/v1/rankings/expert-hit-rate
  ?minSamples={minSamples}
  &page={page}
  &limit={limit}

GET /api/v1/rankings/industry/{industry}
  ?sortBy={sortBy}  // 'excess_return' | 'hit_rate'
  &minSamples={minSamples}
  &page={page}
  &limit={limit}

GET /api/v1/rankings/knowledge-contribution
  ?sortBy={sortBy}  // 'count' | 'views' | 'likes'
  &page={page}
  &limit={limit}
```

## 5. 왜곡 방지 및 신뢰도 확보 방안

### 5.1. 종합 점수 미제공 원칙

초기 버전에서는 종합 점수(통합 랭킹)를 제공하지 않는다. 6개의 기준별 랭킹을 독립적으로 제공하여 사용자가 다양한 각도에서 정보를 평가할 수 있도록 한다. 종합 점수는 기준별 가중치 설정의 자의성 문제와 특정 기준에 의한 왜곡 가능성이 있으므로, 데이터가 충분히 축적되고 사용자 피드백이 수렴된 이후에 도입을 검토한다.

### 5.2. 표본 수 표시 의무화

모든 전문가 랭킹(평균 초과수익률, 적중률, 산업별)에는 반드시 표본 수(검증된 발언 수)를 함께 표시한다. 이를 통해 소수의 운 좋은 발언으로 상위에 랭크된 전문가와, 다수의 검증된 발언으로 꾸준한 성과를 낸 전문가를 사용자가 구별할 수 있다.

### 5.3. 최소 표본 수 기준

| 랭킹 종류 | 최소 표본 수 | 근거 |
|-----------|-------------|------|
| 발언 수익률 TOP | 제한 없음 | 개별 발언이므로 |
| 발언 초과수익률 TOP | 제한 없음 | 개별 발언이므로 |
| 전문가 평균 초과수익률 | 5개 | 통계적 최소 의미 확보 |
| 전문가 적중률 | 5개 | 통계적 최소 의미 확보 |
| 산업별 랭킹 | 3개 | 산업 특화성 고려 |
| 지식 기여 랭킹 | 1개 | Knowledge Note 1개만으로 기여 인정 |

### 5.4. 표본 수 레벨에 따른 시각적 구분

| 표본 수 | 신뢰도 | 표시 방식 |
|---------|--------|-----------|
| 20개 이상 | 높음 | 🟢 또는 신뢰도 바 100% |
| 10-19개 | 중간 | 🟡 또는 신뢰도 바 60% |
| 5-9개 | 낮음 | 🟠 또는 신뢰도 바 30% |
| 3-4개 (산업별) | 매우 낮음 | 🔴 또는 신뢰도 바 15% |

### 5.5. 이상치 탐지 및 제외

다음 조건에 해당하는 발언은 랭킹 산출 시 제외하거나 별도 표시한다.

- 검증 기간 내 발생한 기업 공시, M&A, 상장폐지 등 외부 이벤트로 인한 급등/급락
- 유동성 극히 낮은 종목(거래량 급감)에서 발생한 비정상 수익률
- 발언 후 7일 이내에 반대 방향으로 급변한 경우 (방향성 논란)

## 6. 초기 구현 범위 (MVP)

### 6.1. 포함 항목

| 랭킹 종류 | MVP 포함 | 사유 |
|-----------|----------|------|
| 발언 수익률 TOP | ✅ | 핵심 기능, 구현 난이도 낮음 |
| 발언 초과수익률 TOP | ✅ | 핵심 기능, 구현 난이도 낮음 |
| 전문가 평균 초과수익률 TOP | ✅ | 전문가 검증 핵심 기능 |
| 전문가 적중률 TOP | ❌ (Phase 2) | 적중 기준 정책 결정 필요 |
| 산업별 랭킹 | ❌ (Phase 2) | 산업 분류 체계 정립 필요 |
| 지식 기여 랭킹 | ❌ (Phase 2) | Knowledge Note 기능 완성 후 |

### 6.2. Phase 2 항목

- 전문가 적중률 TOP: 적중 기준(hit/partial_hit/miss)에 대한 구체적인 정책 수립 후 개발
- 산업별 전문가 랭킹: 종목-산업 매핑 데이터 정비 후 개발
- 지식 기여 랭킹: Knowledge Note CRUD 기능 완료 후 개발

### 6.3. 기술 스택 (프론트엔드)

- **프레임워크**: React (Next.js) 또는 기존 프로젝트 프레임워크 준용
- **테이블 라이브러리**: TanStack Table (React Table v8) — 필터, 정렬, 페이지네이션 내장
- **차트 라이브러리**: Recharts 또는 Nivo — 전문가 성과 추이 시각화 (Phase 2)
- **상태 관리**: React Query (TanStack Query) — 서버 데이터 캐싱 및 자동 갱신
- **스타일링**: Tailwind CSS 또는 CSS Modules

## 8. 성능 및 캐싱 전략

### 8.1. 데이터 캐싱

| 계층 | 기술 | 전략 |
|------|------|------|
| 데이터베이스 | PostgreSQL + Redis | 실시간 데이터는 DB 조회, 집계 데이터는 Redis 캐싱 |
| API | Redis Cache (TTL 5분-1시간) | 랭킹 종류별 차등 TTL 적용 |
| 프론트엔드 | React Query (staleTime 30초-5분) | 사용자 인터랙션 고려한 적절한 staleTime 설정 |

### 8.2. 실시간 업데이트

발언 수익률/초과수익률 랭킹은 검증 완료 시점에 실시간으로 업데이트된다. 이를 위해 웹소크(WebSocket) 또는 Server-Sent Events(SSE)를 통한 푸시 알림을 고려한다. 단, 초기 MVP에서는 페이지 새로고침 방식으로 대체 가능.

### 8.3. 데이터 양 증가 대비

랭킹 데이터가 증가함에 따라 다음과 같은 대응 방안을 마련한다.

| 단계 | 데이터 건수 | 대응 방안 |
|------|------------|-----------|
| 초기 | 1만 건 미만 | DB 인덱스 최적화, 기본 페이지네이션 |
| 성장 | 1만-10만 건 | Redis 캐싱 도입, 집계 테이블(Pre-aggregated table) 도입 |
| 대규모 | 10만 건 이상 | Elasticsearch 도입 검토, CDC(Change Data Capture) 기반 실시간 집계 |

## 9. 운영 및 모니터링

### 9.1. 로깅

- 랭킹 페이지 조회 수, 필터 변경 로그, 클릭 이벤트 로그 수집
- 사용자 행동 분석을 통한 랭킹 기준 선호도 파악

### 9.2. 알림

- 특정 전문가의 급격한 순위 변동(예: 10계단 이상 상승/하락) 시 모니터링 알림
- 이상치 탐지 조건에 부합하는 발언 발생 시 관리자 알림

### 9.3. A/B 테스트 계획

| 테스트 항목 | 변수 | 목표 |
|------------|------|------|
| 랭킹 기본 정렬 기준 | 초과수익률 vs 적중률 | 사용자 선호도 및 체류 시간 측정 |
| 표본 수 표시 방식 | 수치 표기 vs 그래픽 표기 | 사용자 이해도 및 피드백 |
| 산업 필터 위치 | 상단 탭 vs 사이드바 | 클릭률 및 사용성 비교 |
