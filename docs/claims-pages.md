# Claims 목록/상세 화면 설계 문서

> **Issue**: #10  
> **목적**: 개별 발언(Claim)을 출처, 구간, 성과와 함께 추적하는 화면 설계  
> **상태**: 초안 (Plan)

---

## 1. 개요

Claims(발언/주장) 화면은 "FM-Stock" 서비스의 검증 데이터 중심 페이지입니다.  
전문가/미디어의 종목 관련 발언을 수집하여, 그 출처와 근거를 투명하게 공개하고,  
이후 실제 주가 흐름과 비교하여 검증 결과를 제공합니다.

이 문서는 두 가지 주요 화면을 정의합니다:

| 화면 | 설명 |
|------|------|
| **Claims 목록** | 모든 발언을 필터링/정렬/검색 가능한 리스트 |
| **Claims 상세** | 개별 발언의 전체 맥락과 검증 결과를 보여주는 상세 페이지 |

## 2. Claims 목록 화면

### 2.1 목록 필터 기준

사용자는 아래 기준으로 발언 목록을 필터링할 수 있습니다.

| 필터 | 타입 | 설명 |
|------|------|------|
| 발언자 | 텍스트 | 발언자 이름 (부분 일치) |
| 종목 | 텍스트 | 종목명 또는 티커 (부분 일치) |
| 산업 | 선택 | 섹터/산업 분류 |
| 방향성 | 선택 | bullish / bearish / neutral |
| 출처 유형 | 다중 선택 | youtube, report, broadcast, news, filing |
| 발언일 | 날짜 범위 | 발언이 이루어진 날짜 범위 |
| 검증 상태 | 선택 | pending / verified / failed |
| 수익률 범위 | 숫자 범위 | 검증 기간 동안의 수익률(%) |
| 초과수익률 범위 | 숫자 범위 | 벤치마크 대비 초과수익률(%) |
| 결과 판정 | 다중 선택 | hit, partial_hit, miss, invalid, pending |

### 2.2 목록 카드/테이블 필드

각 Claim은 다음 정보를 포함하여 표시됩니다.

| 필드 | 설명 | 예시 |
|------|------|------|
| 발언일 | 발언이 이루어진 날짜 | 2025-12-15 |
| 발언자 | 발언한 사람/기관 | 김OO 애널리스트 |
| 종목명 | 언급된 종목 이름 | 삼성전자 |
| 티커 | 종목 코드 | 005930 |
| 산업 | 종목의 산업 분류 | 반도체 |
| 방향성 | 발언의 방향 (상승/하락/중립) | bullish |
| 발언 요약 | 발언의 핵심 내용 (최대 2줄) | "HBM 시장 점유율 확대 예상" |
| 출처 제목 | 출처의 제목 | "삼성전자 HBM 경쟁력 분석 리포트" |
| 구간/페이지 | 발언의 위치 (시간 또는 페이지) | 12:34 ~ 15:20 |
| 기준가 | 기준 시점의 주가 | 72,500원 |
| 검증가 | 검증 시점의 주가 | 85,200원 |
| 수익률 | 기준가 대비 검증가 수익률 | +17.5% |
| 초과수익률 | 벤치마크 대비 초과 수익률 | +8.2% |
| 결과 판정 | 발언의 적중 여부 | hit |

### 2.3 목록 UI 동작

- **기본 정렬**: 발언일 내림차순 (최신순)
- **1페이지 표시 개수**: 20개
- **무한 스크롤** 또는 **페이지네이션** 지원
- **카드 뷰** (기본) / **테이블 뷰** (토글 가능)
- **목록 아이템 클릭** 시 상세 화면으로 이동

## 3. Claims 상세 화면

상세 화면은 5개 섹션으로 구성됩니다.

### 3.1 발언 요약 (Claim Summary)

발언에 대한 핵심 정보를 상단에 표시합니다.

| 필드 | 설명 |
|------|------|
| 발언자 | 발언자 이름 (링크: 발언자 프로필) |
| 발언일 | YYYY-MM-DD 형식 |
| 종목 | 종목명 (링크: 종목 상세) 및 티커 |
| 산업 | 종목의 산업 분류 |
| 방향성 | bullish / bearish / neutral (+ 아이콘) |
| 핵심 발언 문장 | 발언의 핵심 내용 인용문 |
| claimType | forecast(예측) / recommendation(추천) / analysis(분석) / opinion(의견) |

### 3.2 출처 정보 (Source Information)

발언이 발견된 출처의 상세 정보를 제공합니다.

| 필드 | 설명 |
|------|------|
| sourceTitle | 출처의 제목 (예: 유튜브 영상 제목, 리포트 제목) |
| sourceType | 출처 유형: youtube / report / broadcast / news / filing |
| publisher | 발행인/채널/언론사 |
| URL | 출처 원본 링크 |
| 시작 시간 | 유튜브/방송: 시작 시간 (초 단위) |
| 끝 시간 | 유튜브/방송: 끝 시간 (초 단위) |
| 페이지 | 리포트/문서: 페이지 번호 |
| 원본 이동 버튼 | 클릭 시 출처 원본으로 이동 |

**특수 처리:**
- **유튜브**: 시작 시간 링크 생성 (```?t=312s``` 형태)
- **리포트**: 페이지 번호 + URL 조합으로 바로가기

### 3.3 근거와 논리 (Rationale & Logic)

발언자가 제시한 근거와 관련 변수를 체계적으로 정리합니다.

| 필드 | 설명 |
|------|------|
| 발언자 제시 근거 | 발언자가 직접 언급한 근거 요약 |
| 산업 변수 | 해당 산업에 영향을 미치는 변수들 |
| 기업 변수 | 해당 기업에 특화된 변수들 |
| 리스크 | 발언자가 언급한 리스크 요소 |
| 전제 조건 | 발언이 유효하기 위한 전제 조건 |

### 3.4 검증 결과 (Verification Result)

발언의 실제 검증 결과를 수치와 함께 제공합니다.

| 필드 | 설명 |
|------|------|
| 기준일 | 검증 기준 시점의 날짜 |
| 기준가 | 기준 시점의 주가 |
| 검증일 | 검증 시점의 날짜 |
| 검증가 | 검증 시점의 주가 |
| 1M 수익률 | 1개월 후 수익률 (%) |
| 3M 수익률 | 3개월 후 수익률 (%) |
| 6M 수익률 | 6개월 후 수익률 (%) |
| 12M 수익률 | 12개월 후 수익률 (%) |
| 벤치마크 수익률 | 동기간 벤치마크 지수 수익률 (%) |
| 초과수익률 | 알파 (수익률 - 벤치마크) (%) |
| 결과 판정 | hit / partial_hit / miss / invalid / pending |
| 검증 메모 | 검증 과정에서의 특이사항 메모 |

**결과 판정 기준:**
- **hit**: 발언 방향대로 움직이고, 목표치 달성
- **partial_hit**: 방향은 맞았으나 목표치 미달
- **miss**: 발언 방향과 반대 움직임
- **invalid**: 데이터 부족 등으로 검증 불가
- **pending**: 아직 검증 기간이 종료되지 않음

### 3.5 연결된 지식 노트 (Linked Knowledge Notes)

같은 source/segment에서 파생된 knowledge_note를 표시합니다.

| 필드 | 설명 |
|------|------|
| 지식 노트 제목 | 연결된 knowledge_note 제목 |
| 생성일 | knowledge_note 생성일 |
| 요약 | knowledge_note 핵심 내용 |
| 연결 이동 버튼 | knowledge_note 상세 페이지로 이동 |

이 섹션은 발언-지식 연결 구조를 시각화하여,  
같은 출처에서 도출된 여러 인사이트를 한눈에 파악할 수 있게 합니다.

## 4. API 연동 규격 (제안)

### 4.1 Claims 목록 API

```
GET /api/v1/claims
  ?speaker=김OO
&ticker=005930
&industry=반도체
&direction=bullish
&source_type=youtube,report
&date_from=2025-01-01
&date_to=2025-12-31
&verification_status=verified
&profit_rate_min=-10
&profit_rate_max=50
&alpha_min=-5
&alpha_max=30
&result=hit
&page=1
&per_page=20
&sort=date_desc
```

응답: Paginated list of ClaimSummary DTO

### 4.2 Claims 상세 API

```
GET /api/v1/claims/:claim_id
```

응답: ClaimDetail DTO (모든 섹션 데이터 포함)

## 5. 데이터 모델 (제안)

```python
class Claim:
    id: str
    speaker: str
    date: datetime
    stock_name: str
    ticker: str
    industry: str
    direction: str  # bullish / bearish / neutral
    claim_type: str  # forecast / recommendation / analysis / opinion
    summary: str
    source: Source
    rationale: Rationale
    verification: Verification
    linked_notes: list[NoteRef]
```

```python
class Source:
    title: str
    source_type: str
    publisher: str
    url: str
    start_time: int | None
    end_time: int | None
    page: str | None
```

```python
class Rationale:
    reasoning: str
    industry_vars: list[str]
    company_vars: list[str]
    risks: list[str]
    prerequisites: list[str]
```

```python
class Verification:
    base_date: datetime
    base_price: float
    verify_date: datetime
    verify_price: float
    profit_1m: float | None
    profit_3m: float | None
    profit_6m: float | None
    profit_12m: float | None
    benchmark_profit: float
    alpha: float
    result: str
    memo: str
```

## 6. 향후 고려 사항

- Claims 화면에서 직접 검증 요청 기능 (재검증 트리거)
- Claims 통계 대시보드 (발언자별 hit rate, 방향성별 성공률 등)
- Claims 공유 기능 (외부 링크 생성)
- 실시간 발언 알림 (새로운 Claim 등록 시)
- Claims 히스토리 차트 (시간별 발언과 주가 흐름 비교)
- 다국어 지원 (영문/일문 발언 처리)
- Claims 비교 화면 (동일 종목 여러 발언 비교)
