# AI 추출 보조 설계 (AI Extraction Plan)

> **Issue:** #15  
> **작성일:** 2026-05-12  
> **상태:** 계획 (Plan)

---

## 1. 개요 (Overview)

AI 추출 보조 기능은 유튜브 영상 자막 및 증권사 리포트 텍스트에서 **Claim(주장) 후보**와 **Knowledge Note(지식 노트) 후보**를 자동으로 추출하여 사용자의 리서치 업무 효율을 높이는 것을 목표로 한다.

### 1.1 핵심 원칙

- **원본 저장/재배포 목적 아님**: AI는 원본 콘텐츠를 저장하거나 재배포하지 않으며, 추출된 구조화된 정보만을 제공한다.
- **검수 전제**: AI 추출 결과는 항상 Review 화면에서 사용자 검수를 필수로 한다.
- **보조 역할**: AI는 완전한 자동화가 아닌 사용자의 판단을 보조하는 역할에 충실한다.

### 1.2 주요 용어 정의

| 용어 | 설명 |
|------|------|
| Claim (주장) | 특정 종목/산업에 대한 검증 가능한 발언 (목표가, 전망, 분석 등) |
| Knowledge Note (지식 노트) | 교육적/설명적 성격의 산업 또는 기업 정보 |
| Source Summary | 입력 콘텐츠 전체에 대한 주제 요약 |
| Expert (전문가) | 발언자 정보 (이름, 유형, 소속) |
| Confidence In Extraction | 추출 신뢰도 (AI 자체 평가 점수) |

## 2. 입력 방식 (Input Methods)

### 2.1 유튜브 영상 입력

사용자가 유튜브 영상에서 Claim/Knowledge 후보를 추출하고자 할 때 다음 정보를 입력한다:

| 필드 | 설명 | 필수 |
|------|------|------|
| 영상 URL | 유튜브 영상의 전체 URL | 필수 |
| 시작 시간 (Start Time) | 관심 구간 시작 시점 (HH:MM:SS) | 권장 |
| 종료 시간 (End Time) | 관심 구간 종료 시점 (HH:MM:SS) | 권장 |
| 발행일 (Published Date) | 영상 업로드 또는 방송 날짜 | 권장 |
| 발언자 (Speaker) | 주요 발언자 이름 (여럿일 경우 분리 기재) | 선택 |
| 사용자 메모/자막 | 사용자가 직접 붙여넣은 자막 또는 요약 노트 | 필수 |

**처리 흐름:**
1. 사용자가 영상 URL과 함께 관심 구간을 지정
2. 사용자가 직접 시청하며 중요한 부분의 자막/메모를 복사하여 입력
3. AI는 제공된 텍스트를 기반으로 Claim/Knowledge 후보 추출
4. 발행일, 발언자 등 메타데이터는 추출 정확도 향상에 활용

### 2.2 리포트 텍스트 입력

증권사 리포트에서 정보를 추출할 때 다음 정보를 입력한다:

| 필드 | 설명 | 필수 |
|------|------|------|
| 리포트 제목 (Report Title) | 리포트의 공식 제목 | 권장 |
| 발행사 (Firm) | 리포트를 발행한 증권사/리서치 기관 | 권장 |
| 애널리스트 (Analyst) | 리포트 작성자 이름 | 권장 |
| 발행일 (Published Date) | 리포트 발행일 | 권장 |
| 페이지 번호 (Page Number) | 인용 구간의 페이지 번호 | 선택 |
| 사용자 본문 (User Text) | 사용자가 붙여넣은 리포트 본문 일부 | 필수 |

**처리 흐름:**
1. 사용자가 리포트 PDF/웹페이지에서 관심 구간 텍스트를 복사
2. 리포트 메타데이터 (제목, 발행사, 애널리스트, 발행일) 함께 입력
3. AI는 텍스트에서 Claim과 Knowledge Note 후보 추출
4. 페이지 번호는 추후 검증 시 출처 추적 용도로 활용

## 3. AI 출력 JSON 형식 (Output Schema)

AI 추출 결과는 다음과 같은 JSON 구조로 반환된다:

```json
{
  "sourceSummary": "전체 주제 요약 (2-3문장)",
  "experts": [
    {
      "name": "발언자 이름",
      "type": "애널리스트 | 전문가 | 진행자 | 기타",
      "organization": "소속 기관 (증권사, 연구소 등)"
    }
  ],
  "claims": [
    {
      "expertName": "해당 Claim의 발언자 이름",
      "assetType": "주식 | 채권 | 원자재 | 환율 | 기타",
      "ticker": "종목 티커 (없으면 null)",
      "companyName": "회사명 (한글/영문)",
      "industry": "산업 분류",
      "direction": "상승 | 하락 | 중립 | 변동성 증가 | 기타",
      "claimText": "원문 발언 내용 (가능한 원문에 가깝게)",
      "evidence": [
        "발언의 근거/사유 1",
        "발언의 근거/사유 2"
      ],
      "baseDate": "의사결정 기준일 (YYYY-MM-DD)",
      "targetDate": "목표 시점 (YYYY-MM-DD, 없으면 null)",
      "targetPrice": "목표 가격 (숫자, 없으면 null)",
      "confidenceInExtraction": 0.85
    }
  ],
  "knowledgeNotes": [
    {
      "industry": "관련 산업",
      "companyName": "관련 회사명 (선택)",
      "topic": "지식 노트 주제",
      "summary": "핵심 내용 요약 (2-3문장)",
      "keyPoints": [
        "핵심 포인트 1",
        "핵심 포인트 2",
        "핵심 포인트 3"
      ],
      "level": "초급 | 중급 | 고급",
      "tags": ["태그1", "태그2", "태그3"]
    }
  ],
  "warnings": [
    "추출 과정에서 발견된 한계사항이나 주의점"
  ]
}
```

### 3.1 필드별 상세 설명

#### sourceSummary
- 입력된 전체 텍스트의 주제를 2-3문장으로 요약
- 향후 검색/분류를 위한 기본 정보 제공

#### experts
- 입력 텍스트에서 식별된 모든 발언자 목록
- type: 애널리스트, 전문가, 진행자, 기타 중 선택
- 동일인이 여러 유형으로 분류될 수 있음 (예: 진행자이면서 전문가)

#### claims
- **expertName**: experts[] 배열에 정의된 이름과 일치해야 함
- **assetType**: 주식, 채권, 원자재, 환율, 기타 중 선택
- **ticker**: 가능한 표준 티커 사용 (없으면 null)
- **companyName**: 회사명은 가능한 정식 명칭 사용
- **direction**: 방향성은 명확한 경우에만 기재, 불명확하면 "기타"
- **claimText**: 원문에 최대한 가깝게 추출하되, 문맥상 필요한 경우 보완
- **evidence**: 발언의 근거를 리스트로 추출 (없으면 빈 배열)
- **baseDate**: 의사결정 기준일 (보고서 발행일 또는 영상 발행일 참조)
- **targetDate**: 목표 시점 (없으면 null, 임의 생성 금지)
- **targetPrice**: 목표 가격 (없으면 null, 임의 생성 금지)
- **confidenceInExtraction**: 0.0 ~ 1.0 사이의 신뢰도 점수

#### knowledgeNotes
- **industry**: 관련 산업 분류 (없으면 "일반")
- **companyName**: 특정 회사 관련 정보면 기재 (일반 산업 설명이면 생략)
- **topic**: 지식 노트 주제 (간결하고 명확하게)
- **summary**: 2-3문장 핵심 요약
- **keyPoints**: 3-5개의 핵심 포인트
- **level**: 대상 독자 수준 (초급/중급/고급)
- **tags**: 관련 태그 (검색/분류용)

#### warnings
- 추출 과정에서 확인된 한계, 불확실성, 주의사항
- 사용자가 Review 시 특별히 확인해야 할 항목

## 4. AI 추출 지침 (Extraction Guidelines)

### 4.1 Claim vs Knowledge Note 구분

| 구분 기준 | Claim | Knowledge Note |
|-----------|-------|---------------|
| 검증 가능성 | 구체적 사실/전망 (검증 가능) | 일반적 설명/교육 정보 |
| 방향성 | 상승/하락/중립 등 방향성 포함 | 방향성 없음 |
| 특정 종목 | 특정 종목/티커 언급 | 산업 전반 설명 |
| 시간 범위 | 목표가/목표일 있음 | 시간 무관 |
| 예시 | "삼성전자 목표가 10만원" | "반도체 사이클은 보통 2년" |

### 4.2 추출 원칙

1. **검증 가능한 발언과 교육용 설명을 분리**하여 각각 Claim과 Knowledge Note로 추출
2. **종목명/티커/산업/방향성**을 명확히 추출 (정보가 불충분하면 해당 필드에 null/null 저장)
3. **목표가/기간이 없으면 임의 생성 금지**, 대신 warnings에 "목표가 정보 없음" 기록
4. **단순 산업 설명/배경 지식**은 Knowledge Note로 처리
5. **발언자가 불명확**하면 expert name을 "unknown"으로 설정하고 warnings에 "발언자 식별 필요" 기록
6. **여러 사람의 발언을 하나의 Claim으로 합치지 않음** (각 발언자는 별도 Claim)
7. **confidenceInExtraction**은 모든 Claim과 Knowledge Note에 포함

### 4.3 신뢰도 점수 기준

| 점수 범위 | 의미 | 상황 예시 |
|-----------|------|-----------|
| 0.9 ~ 1.0 | 매우 높음 | 명확한 발언, 종목/방향성/근거 모두 명확 |
| 0.7 ~ 0.89 | 높음 | 대부분 명확하나 일부 세부정보 추정 |
| 0.5 ~ 0.69 | 보통 | 발언은 명확하나 종목/방향성 추정 필요 |
| 0.3 ~ 0.49 | 낮음 | 발언이 모호하거나 문맥 의존도 높음 |
| 0.0 ~ 0.29 | 매우 낮음 | 사실상 추정에 가까움, 사용자 검증 필수 |

## 5. 알려진 한계 (Known Limitations)

AI 추출 과정에서 다음과 같은 한계가 있으며, 이는 warnings 필드를 통해 사용자에게 전달된다:

### 5.1 발언자 vs 진행자 구분
- 유튜브 영상에서 발언자와 진행자를 구분하는 것은 어려움
- 자막만으로는 누가 말하는지 식별 불가능한 경우 "unknown" 처리
- 해결 방안: 사용자가 발언자 정보를 사전에 제공하거나 Review 시 직접 구분

### 5.2 종목명 vs 회사명 구분
- 동일한 회사라도 다양한 명칭으로 언급 (정식명, 약칭, 별칭)
- 티커가 없는 비상장사의 경우 식별이 어려움
- 해결 방안: AI 추출 시 가능한 모든 명칭을 후보로 제시, Review에서 확정

### 5.3 목표가 vs 현재가 구분
- "10만원"이 목표가인지 현재가인지 문맥에 따라 판단
- 발언 시점의 주가를 알지 못하면 오판 가능
- 해결 방안: baseDate를 기준으로 현재가 확인 로직 보완 (향후)

### 5.4 조건부 발언 vs 단정적 발언 구분
- "만약 ...하면 ...할 것" 같은 조건부 발언을 단정적으로 해석할 위험
- 해결 방안: 조건문이 포함된 경우 claimText에 조건 유지, direction은 "기타" 처리

### 5.5 산업 전망 vs 개별 종목 전망 구분
- "반도체 업황 회복"이 산업 전망인지 특정 종목 전망인지 모호
- 해결 방안: 특정 종목이 언급되지 않으면 Knowledge Note 처리

### 5.6 발행일 vs 기준일 처리
- 리포트 발행일과 분석 기준일이 다를 수 있음 (예: 주말 리포트지만 금요일 데이터 기준)
- 해결 방안: baseDate는 발행일 우선, 사용자 제공 정보가 있으면 우선 적용

## 6. Review 프로세스 (Review Process)

AI 추출 결과는 항상 Review 화면에서 사용자 검수를 필수로 한다.

### 6.1 검수 항목

1. **Experts 검수**
   - 발언자 이름 정확성 확인
   - 발언자 유형 (애널리스트/전문가/진행자) 확인
   - 소속 기관 정확성 확인

2. **Claims 검수**
   - Claim이 원문 발언을 정확히 반영하는지 확인
   - 종목명/티커 정확성 확인
   - 방향성 (상승/하락/중립) 적절성 확인
   - 목표가/목표일 정확성 확인
   - evidence(근거)가 발언 내용과 일치하는지 확인

3. **Knowledge Notes 검수**
   - 지식 노트 주제와 내용의 적절성 확인
   - keyPoints가 핵심을 잘 포착했는지 확인
   - level (초급/중급/고급) 분류 적절성 확인

4. **Warnings 검토**
   - AI가 스스로 식별한 한계사항 검토
   - 추가 주의사항이 있는지 판단

### 6.2 수정 액션

| 액션 | 설명 |
|------|------|
| 승인 (Approve) | AI 추출 결과를 그대로 승인 |
| 수정 (Edit) | 특정 필드 값을 수정 |
| 삭제 (Delete) | 잘못 추출된 Claim/Knowledge Note 삭제 |
| 추가 (Add) | 누락된 Claim/Knowledge Note 수동 추가 |
| 재추출 (Re-extract) | 입력 수정 후 AI 재추출 요청 |

## 7. 기술 스택 및 확장 계획

### 7.1 현재 범위
- Python 기반 AI 추출 모듈
- JSON 형식의 구조화된 출력
- 사용자 입력 기반 반자동 추출

### 7.2 향후 확장 고려사항
- 자동 자막 다운로드 기능 (youtube-dl 등 활용)
- PDF 리포트 자동 파싱
- 다국어 지원 (영문, 일문 리포트)
- 추출 결과 버전 관리
- Confidence 점수 피드백 루프 (사용자 수정 반영한 모델 개선)

## 8. 결론

AI 추출 보조 기능은 유튜브 영상과 리포트 텍스트에서 Claim과 Knowledge Note를 구조화된 JSON 형식으로 추출하여 사용자의 리서치 업무를 효율화한다. 단, AI 결과는 항상 Review 화면에서 사용자 검수를 필수로 하며, 알려진 한계점을 인지하고 사용해야 한다. 본 계획은 지속적으로 업데이트되며, 실제 구현 과정에서 발견되는 개선점을 반영한다.
