# 자료 수집 정책 (Collection Policy)

> **Refs:** #3  
> **목적:** 백테스트형 금융 자료 수집 기준 및 처리 흐름 정의  
> **범위:** 공개 유튜브 영상, 방송 클립, 증권사 리포트, 투자자 인터뷰, 산업 전망 영상, 종목 분석 영상

---

## 1. 수집 방향

### 1.1 시간 기준
- **6개월 ~ 1년 전** 자료를 우선 수집한다.
- 최신 자료(발행 후 1개월 미만)는 백테스트 기간에 포함되지 않으므로 우선순위에서 제외한다.
- 단, 장기적인 관점에서 분석할 가치가 있는 자료(예: 연간 산업 전망)는 예외로 한다.

### 1.2 자료 종류 (포함 기준)
| 유형 | 설명 | 예시 |
|------|------|------|
| 유튜브 영상 | 종목 분석, 실적 리뷰, 산업 전망 영상 | 삼성전자 2023년 실적 분석, 2차전지 산업 전망 |
| 방송 클립 | 공개된 TV/라디오 방송 클립 | 머니투데이방송 종목토론, 한국경제TV |
| 증권사 리포트 | 애널리스트 공식 보고서 | SK하이닉스, HBM 수요 증가로 목표가 상향 |
| 투자자 인터뷰 | 유명 투자자/애널리스트 인터뷰 | 한국투자증권 김OO 애널리스트 인터뷰 |
| 산업 전망 영상 | 특정 산업군 분석 영상 | 2024년 반도체 산업 전망, K-푸드 산업 분석 |
| 종목 분석 영상 | 개별 종목 심층 분석 | 현대차 투자포인트 3가지 |

### 1.3 우선순위 기준

아래 조건을 **모두 충족**하는 자료를 최우선으로 수집한다.

1. **특정 종목/산업**이 명확히 언급된 자료
2. **실적/목표가/투자포인트/리스크**를 구체적으로 제시한 자료
3. **발행인(채널/애널리스트)** 의 신뢰도가 확인 가능한 자료
4. **백테스트가 가능한 시점**(6개월~1년 전)의 자료
5. 형식적 요건을 충족한 자료 (URL, 제목, 발행일 등 메타데이터 완비)

## 2. 자료별 저장 기준

### 2.1 유튜브 영상

| 항목 | 필수 | 설명 |
|------|------|------|
| URL | ✅ | 영상 고유 주소 |
| 제목 | ✅ | 영상 원제목 |
| 채널명 | ✅ | 업로드 채널 이름 |
| 발행일 | ✅ | 영상 업로드 날짜 |
| 시작/끝 시간 | ✅ | 분석 대상 구간 (타임스탬프) |
| 발언자 | ✅ | 해당 구간 발언자 식별 정보 |
| 관련 종목 | ✅ | 언급된 종목명 (종목코드 포함 가능) |
| 원본 다운로드 | ❌ | 금지 (링크만 저장) |

### 2.2 증권사 리포트

| 항목 | 필수 | 설명 |
|------|------|------|
| 제목 | ✅ | 리포트 제목 |
| 발행사 | ✅ | 증권사/리서치사 |
| 애널리스트 | ✅ | 작성자 이름 |
| 발행일 | ✅ | 리포트 발행일 |
| URL / privatePath | ✅ | 공개 링크 또는 내부 경로 |
| 페이지 번호 | ✅ | 참조 페이지 |
| 원문 PDF 공개 | ❌ | 금지 (링크/경로만 저장) |

### 2.3 기타 자료 (방송 클립, 인터뷰, 산업 전망 등)
- 가능한 한 유튜브 영상 기준에 준하여 저장한다.
- 출처가 명확하지 않은 자료는 수집 대상에서 제외한다.

## 3. 데이터 모델: Source와 Segment 분리

### 3.1 Source (원천 자료)
- **개념:** 하나의 독립적인 자료 단위 (유튜브 영상 1개, 리포트 1건)
- **저장 정보:** URL, 제목, 발행인, 발행일, 자료 유형, 메타데이터

### 3.2 Segment (구간)
- **개념:** Source 내에서 분석 대상이 되는 특정 구간
- **저장 정보:** 시작/끝 시간(영상의 경우), 페이지 번호(리포트의 경우), 발언자, 관련 종목, 태그
- **관계:** 하나의 Source는 여러 개의 Segment를 가질 수 있음 (1:N)

```
Source (1) ---- Segment (1)
             +-- Segment (2)
             +-- Segment (3)
```

## 4. 제외 기준

아래에 해당하는 자료는 수집 대상에서 **제외**한다.

| 제외 사유 | 상세 설명 |
|-----------|-----------|
| **발행 후 1개월 미만** | 백테스트 기간에 부합하지 않음 (예외: 장기 전망 자료) |
| **실시간 리딩성 콘텐츠** | 실시간 방송, 단타 추천, 미공개 정보 전달성 콘텐츠 |
| **근거 없는 루머** | 확인되지 않은 M&A, 실적 전망, 재료 등 |
| **검증 불가능 발언** | 발언자 신원/전문성을 확인할 수 없는 경우 |
| **단순 시황 잡담** | 특정 종목/산업/실적/목표가/리스크가 언급되지 않은 일반 시황 |
| **광고성 콘텐츠** | 특정 종목 매수 유도, 유료방송 홍보 등 |
| **저작권 침해 우려 자료** | 무단 전재, 재가공 등 명백한 저작권 위반 자료 |

## 5. 수집 후 처리 흐름

```
 1 Source 등록
    |
 2 Segment 지정
    |
 3 Claim 추출
    |
 4 knowledge_note 추출
    |
 5 검수
    |
 6 저장
    |
 7 Evaluation
    |
 8 랭킹 반영
```

### 각 단계 설명

| 단계 | 설명 | 담당 (예시) |
|------|------|------------|
| **1 Source 등록** | 수집한 자료를 Source 모델로 등록 (URL, 메타데이터) | 수집 에이전트 |
| **2 Segment 지정** | 분석 구간 식별 및 Segment 모델 생성 | 수집 에이전트 |
| **3 Claim 추출** | Segment 내 주장/사실/의견을 Claim 단위로 추출 | 추출 에이전트 |
| **4 knowledge_note 추출** | Claim을 바탕으로 지식 노트 생성 | 추출 에이전트 |
| **5 검수** | 추출 결과 검증 (Claim의 사실 여부, 출처 일치도) | 검수자/에이전트 |
| **6 저장** | 검수 완료된 데이터 저장 | 시스템 |
| **7 Evaluation** | 자료의 정확성, 유용성, 신뢰도 평가 | 평가 에이전트 |
| **8 랭킹 반영** | 평가 결과를 바탕으로 자료/발언자 랭킹 업데이트 | 시스템 |

### 처리 원칙
- **단계별 승인:** 각 단계는 이전 단계가 완료되어야 진행 가능
- **재처리:** 검수에서 문제 발견 시 3 또는 4 단계로 환류
- **추적성:** 모든 단계에서 원본 Source와 Segment를 참조할 수 있어야 함

## 6. 메타데이터 스키마 (초안)

### 6.1 Source
```json
{
  "sourceId": "uuid",
  "type": "youtube|report|clip|interview|industry|stock",
  "url": "...",
  "title": "자료 제목",
  "publisher": "채널명/발행사",
  "author": "발언자/애널리스트",
  "publishedAt": "YYYY-MM-DD",
  "collectedAt": "YYYY-MM-DDTHH:mm:ssZ",
  "tags": ["태그1", "태그2"],
  "status": "registered|segmenting|claim_extracted|reviewed|evaluated"
}
```

### 6.2 Segment
```json
{
  "segmentId": "uuid",
  "sourceId": "uuid",
  "startTime": "00:01:30" | null,
  "endTime": "00:05:45" | null,
  "pageNumber": 3 | null,
  "speaker": "발언자",
  "content": "구간 내용 요약",
  "relatedStocks": ["005930", "000660"],
  "tags": ["실적", "목표가"]
}
```

### 6.3 Claim (참고)
```json
{
  "claimId": "uuid",
  "segmentId": "uuid",
  "sourceId": "uuid",
  "statement": "주장 내용",
  "claimType": "fact|opinion|prediction",
  "confidence": 0.85
}
```
